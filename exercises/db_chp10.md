6. 불필요한 데이터를 함께 삽입하지 않으면 필요한 데이터를 삽입하지 못하는 현상: **삽입이상**

10. 어떤 릴레이션 R이 2NF이면서, 기본키가 아닌 모든 속성이 기본키에 이행적 함수적 종속이 아니라는 제약조건을 만족하는 정규형: **제삼정규형**

11. 하나의 종속 관계가 하나의 릴레이션으로 표현되도록 릴레이션을 분해하는 과정: **정규화**

## 연습문제

**1. 다음 중 이상 현상에 대한 설명이 잘못된 것은?** **답: ②** - 이유: 삽입 이상은 어떤 데이터를 삽입하면 불필요한 데이터도 함께 삽입되는 현상이 아니라, '불필요한 데이터를 함께 삽입하지 않으면 필요한 데이터를 삽입하는 것이 불가능한 현상'입니다.

**2. 이상현상의 원인과 해결 방법에 대한 설명이 잘못된 것은?** **답: ①** - 이유: 이상 현상의 근본 원인은 단순히 성능 향상을 위해 데이터가 중복되었기 때문이 아니라, 속성 간에 존재하는 여러 개의 종속 관계를 하나의 릴레이션으로 표현했기 때문입니다. 

**3. 함수적 종속에 대한 설명이 잘못된 것은?** **답: ②** - 이유: 속성 X의 각 값에 대해 속성 Y의 값이 오직 하나만 연관되면, "속성 Y는 속성 X에 함수적으로 종속된다"고 표현해야 올바릅니다.

**4. ( ) 안에 적합한 용어는?** 결정자인 복합키에 포함되는 속성 가운데 단독으로 결정자 역할을 하는 속성이 없으면, 종속자가 결정자에 ( **완전 함수적 종속** )이 된다.

**5. ( ) 안에 적합한 용어는?** ( **무손실 분해** )이란 정규화를 통해서 릴레이션을 분해했을 때 손실되는 정보가 없어야 한다는 것이다.

**6. ‘도서(ISBN, 제목, 발행일, 저자id, 저자국적)’ 릴레이션의 함수적 종속 관계가 다음과 같이 표현되었다. 이 릴레이션이 만족하는 정규형은 무엇인가? 답에 대한 이유도 설명하시오.** - **답:** 제2정규형 (2NF)
- **이유:** 기본키인 `ISBN`에 `제목`, `발행일`, `저자id` 등 모든 속성이 완전 함수적 종속이므로 제2정규형을 만족합니다. 하지만 기본키가 아닌 `저자id`가 `저자국적`을 결정하는 이행적 함수적 종속(`ISBN` → `저자id` → `저자국적`)이 존재하므로 제3정규형은 만족하지 못합니다.

**7. 역정규화가 필요한 이유에 대해 설명하시오.** - **답:** 정규화 단계가 진행될수록 릴레이션이 분해되어 원하는 정보를 얻기 위해 여러 테이블을 조인(join)해야 하므로 응답 시간이 길어지고 성능 저하가 발생할 수 있습니다. 따라서 빈번하게 수행되는 검색 질의의 수행 속도를 높이기 위해, 데이터 중복을 감수하고서라도 분해된 릴레이션을 통합하는 역정규화가 필요합니다.

**8. 역정규화에 대한 설명으로 잘못된 것은?** **답: ③** - 이유: 릴레이션을 통합하는 작업 자체는 맞으나, 릴레이션 '수'를 줄이는 것 자체가 목적이 아니라 검색 연산의 '성능 향상(응답시간 단축)'을 위해 대가를 치르고 릴레이션을 합치는 것입니다. 

**9. 다음 회원관리 릴레이션은 헬스클럽의 회원 정보와 회원 관리를 담당하는 트레이너 정보가 함께 저장된 릴레이션으로, 현재 다음과 같은 데이터가 저장되어 있다. 이 릴레이션에 포함된 이상 현상이 잘못 설명된 것은?** **답: ②** - 이유: 해당 릴레이션의 기본키는 '회원번호'이므로 새로운 회원을 삽입할 때 담당 트레이너가 지정되지 않더라도(즉, 트레이너 속성에 널 값을 허용한다면) 개체 무결성 제약조건에 위배되지 않아 새로운 회원 정보를 삽입할 수 있습니다.

**10. 9번 문제에 제시된 회원관리 릴레이션의 함수적 종속 관계를 분석하여 다음 표에 정규화하는 과정을 완성하시오.**

| :--- | :--- |

| **대체키** | 없음 |

| **릴레이션 분해** | 회원(회원번호, 회원이름, 연락처, 가입일자, 담당트레이너(FK))<br>트레이너(담당트레이너, 트레이너이름, 트레이너연락처) |
